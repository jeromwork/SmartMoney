# Data And Demo Notes

## История и тики

- Базовый сценарий: терминал брокера сам подкачивает историю после подключения к серверу.
- Для повторяемых тестов можно импортировать файлы в `data/import` и `data/ticks`.
- `scripts/sync-data.ps1` копирует локально подготовленные данные в нужный инстанс терминала.
- История от брокера и импортированные данные намеренно хранятся отдельно.

## Demo счета

- `terminals/mt4-demo` используется только для demo-forward test.
- После первого ручного логина терминал сохраняет параметры подключения внутри своей папки.
- Повторные запуски идут через `scripts/demo-launch.ps1` без влияния на `dev` и `test`.

## Практика

- Для подкачки истории сначала используйте `mt4-dev`.
- Для воспроизводимых прогонов переносите нужные данные в `mt4-test`.
- Demo-инстанс не использовать для бэктестов и массовой подкачки истории.
